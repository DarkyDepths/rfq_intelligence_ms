What this document is
This is the single authoritative reference for implementing the GHI estimation workbook parser inside rfq_intelligence_ms. It contains the parser contract, all three sheet extraction maps, the frozen typed schema (with v1.1 patches applied), all normalization rules, canonical key tables, cross-check definitions, and the full JSON skeleton. Nothing needed for implementation should exist outside this document.
What this document is NOT
This is not code. This is not the implementation blueprint (folder structure, module interfaces, adapter patterns). That is Step 8, which comes after this contract is frozen. This document defines what the parser must produce. Step 8 defines how it is built.

01 Overview & Scope

1.1 Goal

When a workbook is uploaded (workbook.uploaded event), the service must: open the .xls file, verify it matches the known GHI template, read the needed sheets deterministically, produce clean structured JSON, save parser warnings separately, and update the intelligence snapshot.

Hard rule
Excel parsing is deterministic. LLM is not used to read workbook cells. LLM can be used later only to explain or summarize extracted results.

1.6 Hard fail vs. soft warning vs. no silent failure

CategoryConditionsBehavior
Hard fail File cannot be opened; not .xls; required core sheets missing; core identity anchors missing; template clearly does not match Entire parsing run fails. status = "failed".
Soft warning Optional sheets missing; blank cells; formulas resolving to empty; weird rows; Excel error cells; hidden sheets; totals that don't reconcile exactly Parsing succeeds. status = "parsed_with_warnings". Warning recorded.
No silent failure If something is strange, we record it. We do not hide it.

02 Frozen Design Decisions

DecisionRule
Source-of-truth ownershipGeneral owns primary RFQ identity. Bid S owns bid_meta and bid_summary_lines. Top Sheet owns top_sheet_lines and top_sheet_summary.
Mirrors preservedIdentity repeated in Bid S and Top Sheet is kept as mirror data, not discarded.
Deterministic parsing onlyNo LLM is used to parse workbook cells.
Nullable business schemaEven required business fields are typed as nullable. Required-ness is enforced by parser warnings and hard fail rules, not by making the JSON impossible to serialize.
DatesAll business dates: YYYY-MM-DD (ISO).
RatiosAll percent-like values stored as raw ratios. 0.23 means 23%. UI can multiply by 100 if needed.
Zero vs null0 = explicit zero. null = missing / unavailable / not populated.
Computed totals over cell totalsSummary figures (e.g., General item totals) are computed from kept rows, not read from potentially-broken Excel summary cells.
WorkbookReader adapterParser must use a thin abstraction (e.g., get_cell(sheet, row, col)) so switching from xlrd to openpyxl later doesn't rewrite the whole thing.

Top Sheet label dash-prefix rule
Every regular row label has a leading - character (e.g., "-Revenue Normal Work", "-Direct Material"). Summary rows and section headers have no dash. Before canonical key matching: strip the leading - and trim. This also serves as a free row-type signal.

Boolean quirk

Cell C45 ("Kick of Meeting") is type BOOLEAN with value 0. Same normalization: bool false → 0.

Column F quirk

All 63 non-blank cells in column F are TEXT type containing empty strings — Rev-01 is not populated in this workbook. Normalize empty-string TEXT cells to null.

09 Normalization Rules (Frozen)

9.1 Text normalization (all sheets)

For text comparison and canonical matching: trim leading/trailing whitespace → replace newlines and tabs with spaces → collapse repeated spaces to one.

9.2 Label handling per sheet

SheetRule

Top SheetNormalize whitespace, then strip one leading - if present, then trim, then canonical-match.

9.3 Empty text normalization

If cell type is TEXT and value is empty or whitespace-only → serialize as null.

9.4 Numeric normalization

Boolean false0

9.5 Boolean-like field normalization (General flags only)

"No"false

9.6 Date normalization

For .xls: use xlrd.xldate_as_tuple(value, workbook.datemode) → format to YYYY-MM-DD. The datemode parameter is required — Excel 1900 vs 1904 date systems produce different results from the same serial.

Applies to: rfq_identity.inquiry_date (General D9), bid_meta.tentative_po_date (Bid S G6), bid_meta.dated (Bid S I8), identity_mirrors.top_sheet.dated (Top Sheet C7).

9.7 Revision normalization

Fields H4 (General) and C14 (General items) may come as Excel numeric values like 0.0. Rule: if numeric and whole number, convert to integer string. Example: 0.0 → "0".

9.8 Ratio rule

Column G in Bid S and columns D/F in Top Sheet are stored as ratios. 0.215068 means ~21.51%. Store as share_ratio / rev00_ratio. Do not multiply by 100 in the canonical output.
