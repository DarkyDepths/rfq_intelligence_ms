What this document is
The frozen contract for Pack 2. Defines what each extractor produces, the typed schema extensions, cross-checks, and ownership rules. The companion Blueprint v2.1 defines how to build it. Both documents share one canonical envelope shape.
Core rule — Pack 2 enriches Pack 1, never competes
Every new extractor adds data that Pack 1 could not provide. Where values overlap, Pack 2 adds cross-checks, not replacements.
Row-index convention — 1-based Excel rows everywhere
All row references use 1-based Excel row numbers. Zero-based indices appear only inside reader internals.

01 Overview & Scope

1.1 What Pack 2 adds — the three artifact targets

cost_breakdown_profile
EXTENDED
Gets two new sub-blocks: material_decomposition (Mat Break-up) and financial_profile (Cash Flow). Both enrich the existing cost/commercial picture.
boq_profile
NEW TOP-LEVEL
New artifact block: boq_item_details[] + material_price_table[]. B-O-Q owns component-level physical structure — a different dimension from cost or commercial data.
workbook_profile
Unchanged from Pack 1. Identity, item rows, bid metadata stay here. B-O-Q data does not nest here — it has its own block.
Why this envelope shape
cost_breakdown_profile becomes the single block for everything about money — cost lines (Bid S, Top Sheet from Pack 1), material decomposition (Mat Break-up), and cash flow timing (Cash Flow). These are all financial dimensions of the same RFQ.

boq_profile is physical/structural — what each vessel is made of, at the component level. This is a fundamentally different analytical dimension from cost or commercial data, justifying its own top-level block rather than nesting inside workbook_profile.

1.3 financial_profile — partial Pack 2 scope

Explicitly partial
The architecture brief envisions financial_profile eventually holding: revenue, gross margin, payment structure, bank charges, LC/guarantees, and negative cash flow. Pack 2 implements only the Cash Flow slice — phased cash movements, inflow/outflow structure, and negative cash flow indicators. Pack 3+ will extend it additively with new fields. No schema churn.

06 Frozen Typed Schema — v2.1 Envelope Shape

This is the single authoritative envelope shape for Pack 2
Both the Implementation Pack and the Blueprint must produce exactly this structure. Any conflict between documents is resolved by this section.

6.1 WorkbookParseEnvelope — top level

interface WorkbookParseEnvelope {
  rfq_id: string;
  template_name: string;                     // "ghi_estimation_workbook_v1"
  workbook_format: string;
  workbook_file_name: string | null;
  workbook_blob_path: string | null;
  template_match: boolean;
  parsed_at: IsoDateTimeString;
  parser_version: string;                    // "workbook-parser-v2.1"

  workbook_profile: WorkbookProfile;         // Pack 1 only — unchanged
  cost_breakdown_profile: CostBreakdownProfile;  // Pack 1 + Pack 2 (material_decomposition + financial_profile)
  boq_profile: BoqProfile | null;            // NEW TOP-LEVEL — null if B-O-Q sheet missing/failed
  parser_report: ParserReport;
}

6.3 FinancialProfile (Cash Flow slice only)

interface FinancialProfile {
  identity_mirror: CashFlowIdentityMirror;
  cash_flow_lines: CashFlowLine[];
  cash_flow_summary: CashFlowSummary;
}

interface CashFlowIdentityMirror {
  inquiry_no: string | null;
  project_name: string | null;
  client_name: string | null;
  dated: IsoDateString | null;
}

interface CashFlowLine {
  sheet_row: integer;
  canonical_key: string;
  line_kind: "summary" | "detail";
  label_raw: string | null;
  monthly_values: { month_1 through month_12: number | null };
  total_pct: number | null;
  total_sr: number | null;
}

interface CashFlowSummary {
  total_inflow_sr: number | null;
  total_outflow_sr: number | null;
  total_inflow_pct: number | null;
  total_outflow_pct: number | null;
  net_cash_position_final: number | null;
  months_with_data: integer;
  negative_months_count: integer;
  peak_negative_exposure: number | null;
}
